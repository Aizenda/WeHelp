  在軟體技術快速變更的當下，我在選擇學習新技術，或是選擇投入新領域時，我會做幾件事，具體如下:

Q6:關於這份申請網頁，分享一個開發時的技術心得。

A6:

  如您所見，我這份申請網頁，有很多點擊按鈕顯示不同內容的互動設計，這是透過JavaScript操作HTML DOM達成的，接下來我會以問題二的圖片放大為例子，我會分成兩個部分說明，第一部分是我的思考邏輯，第二部分是我會貼上程式碼，詳盡說明我運用了什麼達成了圖片放大的效果， 但在此之前我會先以我的見解簡單說明什麼是HTML DOM。

什麼是HTML DOM?

  HTML DOM 全名是 HTML Document Object Model，HTML DOM 將 HTML 文檔表示為一個樹狀結構，其中每個 HTML 元素（如 <div>, <p>, <a>等）都被視為一個節點，並且允許通過程式語言（如 JavaScript）來動態讀取和修改這些節點。 依據我的理解就是把 Document 轉換成 Object ，讓 JavaScript 能夠透過操作 Object 的方式操作 Document ， DOM 是瀏覽器與 HTML 文件之間的橋樑，JavaScript 必須依賴 DOM這個橋梁來改變或讀取頁面內容， 以下是JavaScript 可以透過 DOM 來完成的事(包括但不限於):

  • 改變頁面的結構
  • 修改元素的屬性
  • 更新元素的內容
  • 處理用戶互動

圖片該如何放大的思考邏輯:

  首先我的頁面上有幾張圖片，有設定圖片大小限制，形成了縮圖的效果，但圖片太小了，站在用戶的立場，會希望圖片可以盡可能大，這樣才能簡單不費眼力的看到圖片的詳細內容，但站在我的立場，我會希望圖片能夠在不引響我的排版的前提下，盡可能詳盡的呈現給用戶， 所以我想到了其他網頁中點擊圖片放大圖片的功能，我就先以用戶的立場分析，用戶要得到放大圖片會有什麼步驟，所以我得到了這幾個步驟:用戶點擊縮圖→圖片放大→用戶點擊大圖外的其他地方，關閉圖片，得到用戶的操作步驟後，我便以工程的角度思考，該怎麼達成。

  我創建了一個 function 可以在 HTML 載入完成後，可以依據圖片帶的特殊屬性找到圖片，並讓圖片處於待機狀態，隨時等待用戶點擊，當用戶點擊後，我要怎麼判斷用戶已經經過點擊了呢?這部分我使用了事件偵測來偵測滑鼠點擊事件， 接著圖片會放大，而圖片放大的過程會有幾個前提條件，第一點需要一個空間能夠放置我放大的完整圖片，第二點修改圖片顯示大小並顯示，第三點第二次點擊偵測關閉圖片，我便依照幾個步驟建立了幾個 function，可以偵測用戶點擊，然後透過 DOM ，創建一個<div>容器用來存放大的原圖， 最後修改圖片上的 CSS 解除圖片大小限制用原圖並加入預先寫好的顯示屬性( 預設display: none;改成了 display: flex;)，來達成放大的效果，最後的最後，用戶第二次點擊，將關閉圖片空間，這是使用 DOM 刪除了顯示屬性( display: flex;)。


程式碼詳解: 


                            document.addEventListener("DOMContentLoaded", () => {

                                const thumbnails = document.querySelectorAll(".img-wp");
                                const modal = document.createElement("div");
                                const img = document.createElement("img");

                                modal.classList.add("modal");
                                modal.appendChild(img);
                                document.body.appendChild(modal);

  這段程式碼做了幾件事，等待 HTML 的文本載入觸發→回呼函式→找到對應 class 屬性的圖片→建立 div與img 標籤；這裡我建立了一個監聽事件用來監聽DOMContentLoaded(HTML文本載入)事件，會這樣做是因為我希望在HTML文本載入完成後，就先把顯示大圖所需要的環境建置完成，而不是在偵測到滑鼠事件時(onclick)再創建環境 ，因為每有一次滑鼠事件，他就會創建一個新的環境，造成不必要的空間與時間複雜度。 




                                const open = (thumbnail) => {
            
                                    const fullSizeUrl= thumbnail.getAttribute("src");
                                    img.src = fullSizeUrl;
                            
                                    img.classList.add("fullSize");
                                    modal.classList.add("show");
                                };
                            
                                const close = () => {
                                    modal.classList.remove("show");
                                    img.classList.remove("fullSize");
                                };
                            
                                modal.addEventListener("click", close);
                            
                                thumbnails.forEach(thumbnail => 
                                    thumbnail.addEventListener("click", () => open(thumbnail))
                                );
                            });

  接下來我建立幾個 function 分別對應用戶的操作，用forEach方法將每張圖片都加上點擊事件偵測，當事件偵測被觸發便會暫存 thumbnail 的值， 這時 open function觸發，然後透過閉包(Closure)的特性， 將 thumbnail 傳出帶入到 open(參數)中；open 接受到參數便會透過 DOM 獲取圖片的連結並在 div 列中加入顯示 CSS img對應的 CSS 是修改圖片長寬屬性，從而達到放大的效果，當用戶再次點擊， modal 偵測到被點擊，便觸發 close function 刪除了顯示的 CSS(show)。

名詞解釋-閉包(Closure):

  正常情況下，兩個不同的 function 是兩個獨立存在的作用域，簡單點說兩者互不干擾，但當函數"記住"了當下的作用域，並函數內部引用函數外部參數時，就形成了一個閉環，就是閉包(Closure)， 以我的 code 為例: 


                                thumbnails.forEach(thumbnail => 
                                    thumbnail.addEventListener("click", () => open(thumbnail))
                                );


  在這段代碼中，thumbnail 是在每次循環時定義的。每個循環會創建一個新的箭頭函數：() => open(thumbnail)，每個事件監聽器都"記住"了自己的 thumbnail，點擊時能準確獲取對應圖片。，即使循環結束，當用戶點擊某個縮圖時，這個箭頭函數仍然能夠"知道"那個當時的 thumbnail，並將其傳遞給 open() 函數，這時open觸發，完成內部引用外部參數，形成閉環。

  我做個比喻就是想像你正在寫一封信，你寫好信的內容後，放進信封裡並貼上郵票。然後，你把信放進郵箱。當郵差送信時，信封和內容還是會保留著，即使你已經不在郵箱旁邊了。在這個比喻中：

  • 箭頭函數就像那封信，保留著當時的thumbnail（信的內容）。
  • thumbnail 就是信的內容"記住"並傳遞給後續的操作。
  • 閉包就是函數（信封）保存了外部環境的變數（信的內容）並能在將來的某個時刻使用它。
  • 點擊事件是郵差，代替我們將信傳送到外部(open)中
