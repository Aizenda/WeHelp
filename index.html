Q4:軟體技術日新月異，如何確定選擇投入的領域是正確有回報的？

A4:

如何選擇投入的領域:
  在軟體技術快速變遷的當下，我在選擇學習新技術，或是選擇投入新領域時，我會做幾件事，具體如下:

  • 觀察就業市場:當一個新興領域或技術正在崛起，在具有一定的市場價值情況下，便會很直觀的反映在就業市場上， 當就業市場規模逐漸龐大時，那表示就業市場有著這樣的技術需求，那麼短時間內的回報以及未來的成長是可以期待的。
  • 技術社群的討論趨勢:如果一個新技術在社群中引起大量討論，且參與者來自不同背景的專業人士，這表示該技術已經吸引了足夠的關注，並且具備了發展的潛力。 觀察像是 GitHub、Stack Overflow 等平台的討論情況，可以幫助我們了解技術的熱度和普及程度。
如何確定選擇投入的領域是正確並有回報的？:
  至於如何確認投入的領域是正確有回報的，這部分我的想法是在還沒真正地投入進去之前，沒有人知道會發生什麼，所以我認為應該要注重在當下，以我自己為例，我想要轉職那麼轉職成功便是我期望的回報， 而轉職過程中的學習的技術對於我來說，就是正確的，就是這些技術的積累，成就我的轉職成功，所以我認為重點不是所選擇的技術的領域，而是這些技術與領域能不能達成我要的回報；而且在技術更迭如此快速的時代， 某些技術在學習過程中可能已經逐漸走向衰退，在這前提下技術選擇反而不重要，而是持續不斷接觸新的領域、新的技術，不斷地跟進，才是在軟體領域站穩腳跟的唯一真理。

Q5:請描述一件產生明顯負面情緒的經歷，如何處理該情緒？

A5:

 產生負面情緒的事:

公司是一個農場，所以會有繁重的種植工作，而我一開始也是應徵農場助理，自然也需要分擔這些外務工作，但後來我有向主管表明，我正在學習網站的相關知識，剛好公司也需要一個對外的電商網站來銷售產品，我便轉調成了農場內的唯一一個資訊人員，主要負責電商官網的開發， 由於任務性質的轉變，我從外務人員變成了辦公室人員，轉折點也是由此開始，由於外務的勞動量真的太大，所以外務的同仁，也會漸漸地開始抱怨說，我為什麼不需要出去分擔外務的工作，這些我並沒有覺得有什麼異樣，對人我依舊笑臉相迎，直到某次， 被主管請去開一個關於網站開發進度的會議。在一間隔音不太好的房間內，隔壁就這麼剛好，就是茶水間，會議進程到一半時，剛好是外務同仁的休息時間，可能他們以為我開會，需要離開公司，認為我不在公司內，便開始討論起了我，他們從一開始的寒暄客套，講到了我調轉內務的事， 再講到我是因為不想做外面比較勞累的工作，最後開始惡言惡語，我都有聽到，這件事讓我很受傷，我明明也很努力，把要所以有不會的東西，好好學起來，幫公司建立一個好的銷售管道，也花了很多時間，與主管什至是總經理開會，也背負了不少壓力，為什麼他們可以這樣隨意的評斷我!!



 如何處理該情緒:

首先，我是一個情緒來的快去得也快的人，所以只要給我一點時間或有什麼東西分散了我的注意，就能很快地脫離低潮的情緒，另外我很常在跟朋友抱怨的時候，講著講著就忽然發現事情並沒有想像的那麼糟糕，也就自然而然地走出來了； 所以綜合以上幾點，我對於負面情緒我會採取以下兩種策略。

  • 尋找情緒的宣洩口:比如，與朋友抱怨，他們能提供不同的觀點和建議，幫助我釐清自己的情緒，並讓我意識到或許問題並不是完全由我造成的。多數時候，將事情說出來能有效地減輕我的負面情緒，而他們提出的不同觀點也能讓我看到問題的多面性。
  • 轉移注意力:比如，運動，透過全神貫注的運動，專注於調整呼吸以及注意自身身體的狀況來轉移大腦的注意力，並且運動會分泌多巴胺等物質，可以有助於放鬆身心，減少壓力，運動完後負面情緒自然會平復不少。

 正視問題，並解決:

這邊提一個與題目無關，但我認為很重要的看法，我認為會產生負面情緒，必然會有一個相對應的問題，所以事情如果在可以處理的範圍內，那正視產生負面情緒的問題，並解決掉問題，也算是一種從根本上解決負面情緒的方法；我描述的產生負面情緒的根本原因是農場人手不足， 造成外務同仁的任務加重，所以我的解決方法是我向主管提出我每天都會撥出2~3小時支援外務，執行後外務同仁對我的埋怨確實小了很多，我再也沒有因為他們開我副本而產生負面情緒；所以從根本上解決問題，也不失為一種處理負面情緒的方法。

A6:

什麼是HTML DOM?

  HTML DOM 全名是 HTML Document Object Model，HTML DOM 將 HTML 文檔表示為一個樹狀結構，其中每個 HTML 元素（如 <div>, <p>, <a> 等）都被視為一個節點， 並且允許通過程式語言（如 JavaScript）來動態讀取和修改這些節點。依據我的理解，就是把 Document 轉換成 Object，讓 JavaScript 能夠透過操作 Object 的方式操作 Document。 DOM 是瀏覽器與 HTML 文件之間的橋樑，JavaScript 必須依賴 DOM 這個橋梁來改變或讀取頁面內容。以下是 JavaScript 可以透過 DOM 來完成的事（包括但不限於）：

  • 改變頁面的結構
  • 修改元素的屬性
  • 更新元素的內容
  • 處理用戶互動

圖片該如何放大的思考邏輯:

  首先我的頁面上有幾張圖片，有設定圖片大小限制，形成了縮圖的效果，但圖片太小了，站在用戶的立場，會希望圖片可以盡可能大，這樣才能簡單不費眼力的看到圖片的詳細內容，但站在我的立場，我會希望圖片能夠在不影響我的排版的前提下，盡可能詳盡的呈現給用戶， 所以我想到了其他網頁中點擊圖片放大圖片的功能，我就先以用戶的立場分析，用戶要得到放大圖片會有什麼步驟，所以我得到了這幾個步驟:用戶點擊縮圖→圖片放大→用戶點擊大圖外的其他地方，關閉大圖，得到用戶的操作步驟後，我便以工程的角度思考，該怎麼達成。

  我創建了一個 function 可以在 HTML 載入完成後，可以依據圖片帶的特殊屬性找到圖片，並讓圖片處於待機狀態，隨時等待用戶點擊，當用戶點擊後，我要怎麼判斷用戶已經經過點擊了呢?這部分我使用了事件偵測來偵測滑鼠點擊事件， 接著圖片會放大，而圖片放大的過程會有幾個前提條件，第一點需要一個空間能夠放置我放大的完整圖片，第二點修改圖片顯示大小並顯示，第三點第二次點擊偵測關閉圖片，我便依照幾個步驟建立了幾個 function，可以偵測用戶點擊，然後透過 DOM ，創建一個<div>容器用來存放大的原圖， 最後修改圖片上的 CSS 移除圖片大小限制用原圖並加入預先寫好的顯示屬性( 預設display: none;改成了 display: flex;)，來達成放大的效果，最後的最後，用戶第二次點擊，將關閉圖片空間，這是使用 DOM 刪除了顯示屬性( display: flex;)。


程式碼詳解: 


                            document.addEventListener("DOMContentLoaded", () => {

                                const thumbnails = document.querySelectorAll(".img-wp");
                                const modal = document.createElement("div");
                                const img = document.createElement("img");

                                modal.classList.add("modal");
                                modal.appendChild(img);
                                document.body.appendChild(modal);

  這段程式碼做了幾件事，等待 HTML 的文本載入觸發→回呼函式→找到對應 class 屬性的圖片→建立 div與img 標籤； 這裡我建立了一個監聽事件用來監聽DOMContentLoaded(HTML文本載入)事件，會這樣做是因為我希望在HTML文本載入完成後，就先把顯示大圖所需要的環境建置完成，而不是在偵測到滑鼠事件時(onclick)再創建環境 ，因為每有一次滑鼠事件，他就會創建一個新的環境，造成不必要的空間與時間複雜度。 




                                const open = (thumbnail) => {
            
                                    const fullSizeUrl= thumbnail.getAttribute("src");
                                    img.src = fullSizeUrl;
                            
                                    img.classList.add("fullSize");
                                    modal.classList.add("show");
                                };
                            
                                const close = () => {
                                    modal.classList.remove("show");
                                    img.classList.remove("fullSize");
                                };
                            
                                modal.addEventListener("click", close);
                            
                                thumbnails.forEach(thumbnail => 
                                    thumbnail.addEventListener("click", () => open(thumbnail))
                                );
                            });

  接下來我建立幾個 function 分別對應用戶的操作，用forEach方法將每張圖片都加上點擊事件偵測，當事件偵測被觸發便會暫存 thumbnail 的值， 這時 open function觸發，然後透過閉包(Closure)的特性， 將 thumbnail 傳出帶入到 open(參數)中，open 接受到參數便會透過 DOM 獲取圖片的連結並在 div 列中加入顯示 CSS img對應的 CSS 是修改圖片長寬屬性，從而達到放大的效果，當用戶再次點擊， modal 偵測到被點擊，便觸發 close function 刪除了顯示的 CSS(show)。

名詞解釋-閉包(Closure):

  正常情況下，兩個不同的 function 是兩個獨立存在的作用域，簡單點說兩者互不干擾；但閉包是指一個函數記住並能夠訪問定義該函數時的作用域中的變數，即使這個作用域已經結束或外部函數已經返回。 簡單來說，閉包允許內部函數繼續訪問外部函數的變數，即便外部函數已經執行結束，以我的 code 為例: 


                                thumbnails.forEach(thumbnail => 
                                    thumbnail.addEventListener("click", () => open(thumbnail))
                                );


  在這段代碼中，thumbnail 是在每次循環時定義的。每個循環會創建一個新的箭頭函數：() => open(thumbnail)，每個事件監聽器都"記住"了自己的 thumbnail，點擊時能準確獲取對應圖片。，即使循環結束，當用戶點擊某個縮圖時，這個箭頭函數仍然能夠"知道"那個當時的 thumbnail，並將其傳遞給 open() 函數，這時open觸發，完成內部引用外部參數，形成閉環。

  我做個比喻就是想像你正在寫一封信，你寫好信的內容後，放進信封裡並貼上郵票。然後，你把信放進郵箱。當郵差送信時，信封和內容還是會保留著，即使你已經不在郵箱旁邊了。在這個比喻中：

  • 箭頭函數就像那封信，保留著當時的thumbnail（信的內容）。
  • thumbnail 就是信的內容"記住"並傳遞給後續的操作。
  • 閉包就是函數（信封）保存了外部環境的變數（信的內容）並能在將來的某個時刻使用它。
  • 點擊事件是郵差，代替我們將信傳送到外部(open)中。
